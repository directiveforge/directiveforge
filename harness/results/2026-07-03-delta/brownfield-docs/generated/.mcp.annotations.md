# MCP & Config Annotations — Trailhead Collective

> Companion to `.mcp.json` and `.claude/settings.json`. Strict JSON cannot carry comments —
> `//` comments break MCP loading. Every posture and rationale lives HERE instead.

## Active servers

**None.** `.mcp.json` ships with an empty `mcpServers` object by design. This is a
pure-Markdown ops repo with no code, no build, no external-service dependencies, and no
`.git` remote — none of the standard dev MCP servers earn their place here:

- **GitHub MCP** — not applicable: this repo has no `.git` and no GitHub remote. If the repo
  is ever put on GitHub, the vetted server is the remote HTTP endpoint
  `https://api.githubcopilot.com/mcp/` (OAuth, no PAT) — see the kit registry.
- **Context7** — skipped: the corpus is business-operations documentation, not library-API
  docs, so there is no outdated-API-hallucination surface for it to cover.
- **Database / service connectors** — none: there is no database and no wired service.

## Governance posture (D4 — binding)

This project gates every external integration. Per decision **D4** and
`governance/INTEGRATIONS-REGISTRY.md`, **no MCP server, credential, or service is wired up
without a founder bootstrap.** The generator therefore PROPOSES only and leaves `.mcp.json`
empty. If a future workflow genuinely needs an MCP server:

1. Draft a candidate row in `governance/INTEGRATIONS-REGISTRY.md` under "Proposed — awaiting
   founder bootstrap": the tool, the job it does, why an existing vetted tool can't cover it,
   the required credential scope (least privilege), and any recurring cost.
2. Stop there — do not create an account, store a key, or add it to `.mcp.json`.
3. The founder bootstraps the credential and moves the row to "Vetted and in use."

A proposal that arrives pre-wired is a governance violation and gets torn out.

## If a server is ever added

- Pin npm/stdio versions (`@x.y.z`, never `@latest`); remote HTTP servers are version-managed upstream.
- `${ENV_VAR}` for every credential — never a literal value in config.
- After any MCP config change: `uvx snyk-agent-scan@latest` (full verdict needs a free `SNYK_TOKEN`).
- Registry of vetted servers + postures: `<KIT_ROOT>/knowledge-base/MCP-SERVER-REGISTRY.md`.
