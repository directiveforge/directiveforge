# MCP & Config Annotations — SlotHarbor

> Companion to `.mcp.json` (and `.claude/settings.json`).
> Strict JSON cannot carry comments — JSONC `//` comments break MCP loading in Claude Code.
> Every server's security posture and rationale lives HERE instead of inline.

## Active servers

### github — remote HTTP (`https://api.githubcopilot.com/mcp/`)

- **Posture**: RW (repos, issues, PRs, code search — 51 tools) via OAuth browser consent; no PAT env var to manage; lockdown mode available against prompt injection
- **Add when**: project is on GitHub and agents need PR/issue access
- **Skip when**: air-gapped (fall back to `gh` CLI via Bash)
- **Note**: the legacy `@modelcontextprotocol/server-github` npm package is DEPRECATED — remote HTTP is the only supported install.

### context7 — `@upstash/context7-mcp@3.2.2` (npm, pinned)

- **Posture**: RO — fetches public documentation only; no credentials
- **Add when**: always — prevents outdated-API hallucinations. Especially load-bearing here: FastAPI + Pydantic v2 + SQLAlchemy 2.0 APIs are frequently hallucinated as their v1/1.x forms.
- **Skip when**: fully offline environments
- **Version**: `3.2.2` is the registry-pinned latest (live-verified 2026-07-03).

### postgres — `mcp-postgresql-ops@3.2.8` (PyPI via `uvx`)

- **Posture**: RO — 30+ read-only tools (schema inspection, EXPLAIN, index/table stats, vacuum monitoring); NO write surface. Credential via `${DATABASE_URL}`.
- **Add when**: agents need to understand the schema or verify a query against the real Postgres instance.
- **Skip when**: pointing at a production DB — use a read replica or a read-only DB user for defense in depth. The kit registry explicitly forbids `@modelcontextprotocol/server-postgres` (archived, SQL-injection CVE) — this Python server is the vetted replacement.
- **Version note**: registry pins `@3.2.8`; PyPI latest is `3.3.7` as of 2026-07-03. Kept at the registry-vetted pin — bump only after re-running the 5-gate quality check.

## Not installed (considered, deliberately skipped)

- **git MCP** (`mcp-server-git`, PyPI) — Claude Code has built-in git access; adding it duplicates tool surface. Skipped per registry guidance.
- **SQLite MCP** — dev uses SQLite, but the postgres ops server + Claude Code's built-in file access cover schema needs; not worth the extra tool surface.

## Security protocol

- Pin npm/stdio server versions (`@x.y.z`, never `@latest`); remote HTTP servers are version-managed upstream
- `${ENV_VAR}` for every credential — literal values in config are a checklist failure
- After any MCP config change: `uvx snyk-agent-scan@latest` (full verdict needs a free `SNYK_TOKEN` from app.snyk.io; without it it degrades to inventory and stdio servers are auto-declined in non-interactive runs)
- Registry of vetted servers + postures: `<KIT_ROOT>/knowledge-base/MCP-SERVER-REGISTRY.md`

## Appendix — `.claude/settings.json` levers

Semantics per `<KIT_ROOT>/workflows/WORKFLOW-CLAUDE-CODE.md` §7.4 / §7.5 / §11.3:

- `MAX_THINKING_TOKENS` — cap ~10000 for routine work; `0` disables thinking entirely
- `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` — `50` for long sessions; compacts before the quality cliff
- `CLAUDE_CODE_SUBAGENT_MODEL` — **omitted** from settings.json for inherit (v2.1.196+); set to `haiku` only if you want mechanical subagents routed cheap
- No format-on-write hook: no formatter (black/ruff) is installed, so `hooks` is `{}`. Add a `PostToolUse Write|Edit` hook if you adopt one.
- Permissions: safe reads (git, ls, grep, wc) pre-allowed; app-boot + alembic + docker build allowed; destructive commands (force-push, hard reset, `.env` reads) denied.
