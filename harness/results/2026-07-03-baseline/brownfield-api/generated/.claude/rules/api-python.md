# API / DB Conventions (app/**)

## FASTAPI
- DO define request/response bodies as Pydantic models per router — never `dict` or raw JSON.
- DO map ORM rows through `response_model` + `ConfigDict(from_attributes=True)` — NEVER return a SQLAlchemy model directly.
- DO inject the DB session with `session: Session = Depends(get_session)` — never construct `SessionLocal()` in a handler.
- DO raise `HTTPException` for expected errors (e.g. 404 dock-not-found, 422 bad window) — never return an error dict.

## SYNC-ONLY (this codebase — DECISIONS #1)
- DO write plain `def` route handlers with a sync `Session`.
- DO NOT introduce `async def` handlers, `AsyncSession`, or `create_async_engine` — the whole DB stack is synchronous. This overrides the generic FastAPI-async default.

## PYDANTIC v2
- DO use `model_config = ConfigDict(...)`, `model_dump()`, `field_validator`.
- DO NOT use v1 `class Config`, `.dict()`, or `@validator` (removed in v2).

## SQLALCHEMY 2.0
- DO query with `select(...)` + `session.scalars()` / `session.execute()`.
- DO NOT use the legacy `session.query()` API.
- DO declare models with `Mapped[...]` + `mapped_column(...)` on `DeclarativeBase` (see `app/models.py`).

## CONFIG
- DO read settings via `from app.config import settings` — never `os.getenv` inline here.

## MIGRATIONS (migrations/**)
- DO read every autogenerated migration before applying — Alembic misses column-type changes and some constraints.
- DO NOT edit `sqlalchemy.url` in `alembic.ini` — the URL is injected from `app.config.settings` in `migrations/env.py`.
