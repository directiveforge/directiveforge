# Tempo Deck

A tiny personal tool for tracking metronome practice sessions — Next.js 15.1 (App Router), React 18.3, TypeScript strict, Tailwind 3.4.

## Critical Constraints

- This is a **solo, pre-MVP side project** — no accounts, no database, no production users. The home page renders a seeded `SEED_SESSIONS` array (`app/page.tsx`); there is no persistence layer. Do NOT invent one without being asked.
- **No test framework is installed.** There is no `test` or `typecheck` npm script. Do not claim tests pass — there are none. `npm run lint` and `npx tsc --noEmit` are the only automated gates (see below).
- `lib/email.ts` is a **dormant stub** — it no-ops unless `RESEND_API_KEY` is set, and no key is configured. Do not wire it into a request path without being asked.
- NEVER commit `.env` / `.env.local` or the `RESEND_API_KEY` value.

## Build & Test Commands

Canonical command table lives in `AGENTS.md` — this section notes only quirks.
- There is **no `typecheck` script**. Use `npx tsc --noEmit` (tsconfig is `strict: true`, `noEmit: true` — it type-checks without emitting).
- There is **no test runner**. `npm run lint` runs `next lint` (ESLint `next/core-web-vitals`).

## Code Style

- Named exports for components (`export function BeatCounter`). Route entry files (`app/page.tsx`, `app/layout.tsx`) additionally `export default` the same function — Next.js requires the default for pages/layouts.
- `@/*` path alias maps to the **project root** (`tsconfig.json` `paths`), so imports read `@/components/beat-counter`, not `@/src/...`. There is no `src/` directory.
- Files are `kebab-case.tsx`; components are `PascalCase`. Brand colors come from `tailwind.config.ts` tokens (`beat`, `stage`, `accent`) — use the tokens, not raw hex.

## Architecture

- **Entry**: `app/` (App Router) — `layout.tsx` wraps, `page.tsx` renders the home route. Server components by default.
- **Rendering**: Server components by default — add `'use client'` only for interactivity (state, events, browser APIs). No client components exist yet.
- **Components**: `components/` — presentational, one component per file, composed top-down (`page` → `session-list` → `beat-counter` → `tempo-badge`).
- **Helpers**: `lib/` — currently just `email.ts` (Resend stub).
- **No API routes, no middleware, no data layer.** Do not assume `app/api/` or `middleware.ts` exist — they do not.
- **Deploy**: Vercel (hobby), framework preset in `vercel.json`, region `iad1`.

## Key Conventions

- Imports use the `@/` alias to root — never relative paths that cross directory boundaries.
- Data is passed as typed props (`PracticeSession`) top-down; no fetching, no global state.
- Read `.env.example` for env var names; never read `.env` / `.env.local`.

## Common Pitfalls

- `@/` resolves to the **repo root**, not `src/` — there is no `src/`.
- Do not add `'use server'` / Server Actions "because Next.js" — nothing here needs them yet.
- Middleware would run on the Edge runtime — no Node built-ins (`fs`, `crypto`) — but there is no `middleware.ts` and none is needed.
- `next/headers`, `cookies()`, and `params` are async in Next.js 15 — `await` them if you ever add dynamic routes.
- Use `next/image` (not `<img>`) if you add images — provide `width` + `height` or `fill`.
