## Active servers

### github — remote HTTP (`https://api.githubcopilot.com/mcp/`)

- **Posture**: RW (repos, issues, PRs, code search) via OAuth browser consent; no PAT env var to manage
- **Add when**: project is on GitHub and agents need PR/issue access
- **Skip when**: air-gapped (fall back to `gh` CLI via Bash)
- **Note for this project**: Tempo Deck has no `.git` yet. This server is harmless until the repo is pushed to GitHub; it consumes a little tool surface. Remove it if you never host on GitHub.

### context7 — `@upstash/context7-mcp@3.2.2` (npm, pinned)

- **Posture**: RO — fetches public documentation only; no credentials
- **Add when**: always — prevents outdated-API hallucinations for Next.js App Router (the API changed significantly across v13–15) and React
- **Skip when**: fully offline environments

## Phase 2.5 discovery — vetted candidates NOT installed

Two external services were detected (`vercel.json` → Vercel; `RESEND_API_KEY` in `.env.example` → Resend). Both have a viable MCP, but neither is wired in for a Starter-tier solo hobby project (operator declined extra MCP servers; "just the basics"). Enable either when the need becomes real.

### Vercel MCP — remote HTTP (`https://mcp.vercel.com`) — READY

- **Registry-vetted** (`MCP-SERVER-REGISTRY.md`, Vercel official). **Posture: RO** via OAuth — deploy health, env-var inspection (read), build-log debugging. No credentials to store.
- **Enable when**: you want deploy monitoring / build-log debugging from the agent. Paste into BOTH configs:

```json
"vercel": {
  "type": "http",
  "url": "https://mcp.vercel.com"
}
```
(Cursor config omits the `"type"` key.)
- **Do NOT** use the community `@vercel/mcp-adapter` + `VERCEL_TOKEN` pattern some presets show — the registry's anti-recommendations flag token-in-config Vercel servers; the official OAuth remote above is the safe choice.

### Resend MCP — `resend-mcp@2.9.0` (npm) — vetted, deferred (Gate 5)

Quality-gate result:
- **Gate 1 EXISTS**: pass — `resend-mcp@2.9.0`, repo `github.com/resend/resend-mcp` (official Resend org), MIT.
- **Gate 2 MAINTAINED**: pass — published 2026-06-23, 23 versions, not deprecated.
- **Gate 3 TRUSTED**: pass — official (maintainers are the Resend team, incl. co-founder zenorocha).
- **Gate 4 SECURE**: not fully verified in this run — `uvx snyk-agent-scan` needs a `SNYK_TOKEN` for a full verdict and auto-declines stdio servers in non-interactive runs. Run it with a token before adopting.
- **Gate 5 USEFUL**: **FAIL (deferred)** — Resend is only a dormant stub here (`lib/email.ts` no-ops without a key, no key configured, no email actually sent). Per the Phase 2.5 decision matrix, an unused service does not justify a write-capable email MCP on a Starter hobby project.
- **Enable when**: you actually wire up recap emails (set `RESEND_API_KEY`, call `sendPracticeRecap` from a real path). Then re-run Gate 4 with a `SNYK_TOKEN` and add with `${RESEND_API_KEY}`:

```json
"resend": {
  "command": "npx",
  "args": ["-y", "resend-mcp@2.9.0"],
  "env": { "RESEND_API_KEY": "${RESEND_API_KEY}" }
}
```

## Security protocol

- Pin npm/stdio server versions (`@x.y.z`, never `@latest`); remote HTTP servers are version-managed upstream
- `${ENV_VAR}` for every credential — literal values in config are a checklist failure
- After any MCP config change: `uvx snyk-agent-scan@latest` (full verdict needs `SNYK_TOKEN`)
- Registry of vetted servers + postures: `<KIT_ROOT>/knowledge-base/MCP-SERVER-REGISTRY.md`

## Appendix — `.claude/settings.json` levers

Semantics per `<KIT_ROOT>/workflows/WORKFLOW-CLAUDE-CODE.md` §11.3:

- `MAX_THINKING_TOKENS` — set to `10000` here (routine work; raise for hard planning)
- `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` — `50` for long sessions; compacts before the quality cliff
- `CLAUDE_CODE_SUBAGENT_MODEL` — **omitted** so subagents inherit the main model (v2.1.196+); set to `haiku` if you want mechanical subagents routed cheap
- `hooks` — `{}` (empty): no formatter is configured (no Prettier/Biome). Add a `PostToolUse Write|Edit` format-on-write hook if you adopt one.
- permissions — pre-allow safe (git reads/commits, build, lint, dev, `npx tsc`); deny destructive (force-push, hard reset, `.env` reads)
