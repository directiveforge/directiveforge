# unitkit

A tiny TypeScript (CommonJS) Node CLI that converts between metric and imperial units — length and mass — using factor-based conversion.

## Build & Test Commands

Full command table lives in `AGENTS.md`. Quirks only:
- No dev server and no `dev` script — it's a CLI. Iterate with `npm run build`, then run `node dist/index.js 5 km in miles`.
- `npm run build` is `tsc`: it compiles AND type-checks. A clean build means types pass — there is no separate typecheck script.
- No test suite yet (DECISIONS #5). Do NOT invoke `npm test` — the script does not exist.

## Architecture

Three modules under `src/` (compiled to `dist/`, CommonJS):
- `units.ts` — `UnitDef` interface + the `UNITS` table (name / aliases / `toBase` factor / `dimension`) + `findUnit()` (case-insensitive name-or-alias lookup).
- `convert.ts` — `ConversionError` + `convert(value, from, to)` = `(value * from.toBase) / to.toBase`, guarded so cross-dimension conversions throw.
- `index.ts` — CLI entry (`bin` → `dist/index.js`); parses `<value> <from> in <to>` and maps outcomes to exit codes (0 ok / 1 ConversionError / 2 usage).

Invariant: every conversion is a single multiplicative factor to the dimension's base unit. Anything needing an offset (temperature) does NOT fit this model.

## Key Conventions

- Named exports only — no default exports anywhere.
- A new unit = one row in the `UNITS` table (`src/units.ts`); never scatter conversion factors across files.
- Failures are thrown as `ConversionError`; `index.ts` is the ONLY place that catches them and maps to exit codes.
- `console.log` → results (stdout); `console.error` → failures (stderr). Keep that split.

## Common Pitfalls

- The 3rd CLI token (the word "in") is IGNORED — `index.ts` destructures `[value, from, , to]`, so `unitkit 5 km to miles` also works. Don't "fix" it without changing that destructure.
- `no-explicit-any` is an ERROR in `.eslintrc.json`, not a warning — never introduce `any`.
- Temperature is in the `dimension` union but has NO units in the table on purpose (needs offset math). Do NOT add temperature rows to the factor table — the factor model would produce wrong results.
- `dist/` is git-ignored, tsc-built output — never hand-edit it; edit `src/` and rebuild.
