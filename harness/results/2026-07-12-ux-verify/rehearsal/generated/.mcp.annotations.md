> Companion to `.mcp.json` (and `.claude/settings.json`).
> Strict JSON cannot carry comments — JSONC `//` comments break MCP loading in Claude Code.
> Every server's security posture and rationale lives HERE instead of inline.

## Security protocol

- Pin npm/stdio server versions (`@x.y.z`, never `@latest`); remote HTTP servers are version-managed upstream.
- `${ENV_VAR}` for every credential — literal values in config are a failure. (unitkit currently has zero credential-bearing servers.)
- After any MCP config change: `uvx snyk-agent-scan@latest` (full verdict needs `SNYK_TOKEN`).
- Registry of vetted servers + postures: `<KIT_ROOT>/knowledge-base/MCP-SERVER-REGISTRY.md`.

## Appendix — `.claude/settings.json` levers

Semantics per `<KIT_ROOT>/workflows/WORKFLOW-CLAUDE-CODE.md` §7.4 / §7.5 / §11.3:

- `MAX_THINKING_TOKENS` / `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` / `CLAUDE_CODE_SUBAGENT_MODEL` — available levers, intentionally left UNSET here (Starter, "just the basics"): thinking budget and subagent model inherit defaults. Add them if you later want to cap thinking (~10000) or route mechanical subagents to `haiku`.
- **Hooks**: `hooks` is `{}` — no formatter (Prettier/Biome) is configured, so there is no format-on-write hook. Add a `PostToolUse` `Write|Edit` hook if you adopt a formatter.
- **Permissions**: pre-allow safe reads + the two real gates (`npm run build`, `npm run lint`) + `npx tsc` (typecheck) + `node dist/index.js` (run the built CLI); deny destructive ops (force-push, hard reset) and `.env` reads.
